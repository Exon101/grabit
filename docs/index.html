Installation

Up and running in 60 seconds.

GrabIt isn't on the Chrome Web Store yet — install it from source. It's a 30-second process.

1

Download the latest release

Grab grabit.zip from the latest GitHub release:

Get the zip
2

Unzip it somewhere permanent

Extract the zip to a folder you won't delete (e.g. ~/Documents/grabit). Chrome needs the folder to stay in place while the extension is installed.

unzip grabit.zip -d ~/Documents/grabit
3

Open Chrome's extensions page

Paste this in your address bar:

chrome://extensions/

Then toggle Developer mode on (top-right corner).

4

Load the extension

Click Load unpacked and select the folder you extracted to. The GrabIt icon appears in your toolbar — pin it for easy access.

5

Visit a supported site

Open a YouTube video, Twitter tweet, or any supported page. Click the GrabIt toolbar icon — you'll be asked to grant host permission for that site (one-time). After granting, the popup shows detected media with quality variants.

How it works

Three layers of fallback.

Many CDNs reject direct downloads without the right Referer header. GrabIt tries three strategies in order.

Tier 1

Direct download

chrome.downloads.download() with the URL as-is. Fastest path. Works for any URL that doesn't enforce Referer.

Tier 2

Fetch + Blob

Service-worker fetch() with cookies, then convert to a Blob URL and pass to chrome.downloads. DNR rules inject Referer/Origin on the network layer.

Tier 3

Open in tab

Last resort: open the URL in a background tab. Chrome will either render the media inline or trigger its own download. We mark it in the recent list as "opened-in-tab".

Privacy

Your data stays on your device.

GrabIt doesn't have a server. It can't see your data.

✓

No telemetry

Zero analytics, zero error reporting, zero third-party requests. The only network calls go directly to the sites you're already browsing.

✓

No third-party servers

Extractors fetch HTML/JSON directly from the site you're on. No proxy, no API gateway, no shared backend.

✓

Cookies never read

Your session cookies are sent with extraction requests so logged-in-only content works — but cookies are never inspected, stored, or transmitted elsewhere.

✓

Settings sync via Google

Your preferences sync across signed-in Chrome instances via chrome.storage.sync — encrypted, tied to your Google account, never visible to us.

✓

Recent downloads are local

The list of recent downloads lives in chrome.storage.local and never leaves your device. Clear it anytime from the popup or options page.

✓

Open source

Every line of code is on GitHub. Audit it, fork it, build it yourself.

FAQ

Common questions.

Why does Chrome ask for permission the first time I use GrabIt on a site?

GrabIt uses optional host permissions — instead of requesting access to all 8 supported sites upfront at install time, it asks only when you first try to use it on a specific site. This keeps your privacy surface minimal: if you never use GrabIt on Facebook, it never has any access to facebook.com. You can revoke a site's permission anytime in Settings → Sites.

Why can't I download some YouTube videos?

YouTube serves many videos with "ciphered" URLs that need JavaScript on the page itself to decode. GrabIt includes a bridge script that resolves many of these, but YouTube changes their decoder frequently. Age-restricted, member-only, and DRM-protected videos also return errors. For those cases, GrabIt shows a friendly message in the popup.

Why are Bilibili video and audio separate?

Bilibili serves media as DASH, with video and audio as separate streams. GrabIt exposes them as separate variants (audio-only and video-only). Downloading the video-only variant yields a silent file. Muxing them into a single MP4 requires ffmpeg.wasm — planned for v1.2.

Does GrabIt work on Firefox, Edge, or Brave?

GrabIt is built for Chrome (Manifest V3). It should also work on Edge, Brave, and other Chromium browsers with minimal changes. Firefox supports MV3 but with some API differences — porting would require a small amount of work.

Why isn't it on the Chrome Web Store?

The Web Store review process for downloaders is slow and frequently rejects extensions that touch YouTube or Facebook. For now, install from source (see the Install section above). A Web Store listing is planned for after v1.2 stabilizes the YouTube extractor.

How do I report a bug or request a feature?

Open an issue on GitHub Issues. Please include the site URL, the version (visible in Settings → About), and any error from the service worker DevTools console (chrome://extensions/ → GrabIt → "Inspect views: service worker").
